# Cotex Search 品牌分析报告前端展示PRD

## 1. 报告整体结构

### 1.1 页面组成
- **页面标题区**: 品牌名称+报告类型+生成日期
- **执行摘要区**: 简要概述品牌表现及核心发现
- **数据指标区**: 关键数据可视化呈现
- **品牌分析区**: 详细分析内容，按模块展示
- **行动建议区**: 针对品牌提供的具体建议
- **竞品推荐区**: 展示值得关注的竞品信息

### 1.2 用户交互
- 支持报告内容导出为PDF/PNG
- 支持分享报告链接
- 支持交互式图表（悬停查看详情）
- 支持原声内容展开/收起
- 支持品牌筛选对比

## 2. 视觉设计规范

### 2.1 配色方案
- **主色调**: #3366FF (品牌蓝)
- **辅助色**: #FF6633 (强调色), #EFEFEF (背景色)
- **数据可视化色**: 使用渐变色方案，确保色彩区分度
- **文字颜色**: 标题 #333333, 正文 #666666, 说明文字 #999999
- **品牌专属色**: 每个品牌使用固定色调，保持一致性

### 2.2 排版规则
- **标题字体**: 18px, 加粗
- **小标题**: 16px, 加粗
- **正文字体**: 14px, 常规
- **说明文字**: 12px, 浅色
- **图表标题**: 15px, 加粗
- **内容间距**: 统一使用8的倍数(8px, 16px, 24px, 32px)

### 2.3 组件样式
- **卡片样式**: 圆角8px, 阴影0 2px 8px rgba(0,0,0,0.1)
- **按钮样式**: 圆角4px, 高度36px
- **分隔线**: 1px, #EEEEEE
- **交互状态**: 悬停时透明度变化或轻微放大
- **强调样式**: 关键数据使用加粗和高亮色标记

### 2.4 用户原声数据要求

#### 2.4.1 原声数据质量标准
- **真实性**: 所有用户原声必须来自真实数据源，严禁使用虚构内容
- **相关性**: 每条原声必须与其支持的洞察点直接相关，不可复用无关原声
- **多样性**: 不同模块和洞察点的原声应当来自不同用户和不同内容
- **时效性**: 优先使用近期内容，反映当前市场情况
- **代表性**: 选择最具代表性的原声，优先选择互动数据(点赞、评论)高的内容

#### 2.4.2 原声数据展示规范
- 每个洞察点至少提供3条相关原声
- 原声必须包含实际内容、来源平台、互动数据(点赞数、评论数)
- 原声URL必须可访问、与内容匹配
- 每个分析模块不得缺少原声支持
- 不同模块间的原声应避免过度重复

## 4. 响应式设计规范

### 4.1 桌面端(>=1200px)
- 双栏布局，重要模块左侧，次要模块右侧
- 图表尺寸最大化展示
- 完整显示所有数据指标
- 允许多品牌同时对比

### 4.2 平板端(768px-1199px)
- 单栏布局，模块依次排列
- 图表自适应缩放
- 部分次要数据默认折叠
- 最多支持三个品牌同时对比

### 4.3 移动端(<=767px)
- 单栏紧凑布局
- 图表简化，优先展示核心数据
- 展开/收起交互增强
- 原声示例默认收起
- 限制为单品牌展示，通过切换比较

### 5.1 页面加载动效

- 数据对比时提供高亮效果
